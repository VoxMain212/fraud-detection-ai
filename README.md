# Система обнаружения мошеннических транзакций

## Описание задачи
Задача бинарной классификации для автоматического определения мошеннических финансовых операций. На вход модели подаются сумма транзакции, время проведения, местоположение и тип устройства. Целевая переменная принимает значение 0 (легитимная операция) или 1 (мошенничество). В проекте используются только классические алгоритмы машинного обучения.

## Данные
Датасет состоит из 10 000 записей в формате CSV. Данные сгенерированы программно с учётом статистических закономерностей реальных финансовых операций и распределения вероятностей мошенничества в зависимости от заданных признаков. Файл создается автоматически при запуске скрипта `create_data.py`.

## Методы машинного обучения
Реализованы и сравнены три алгоритма:
- Logistic Regression
- Random Forest Classifier
- Gradient Boosting Classifier

Предобработка данных включает кодирование категориальных признаков (`LabelEncoder`) и нормализацию числовых значений (`StandardScaler`). Для корректной работы с несбалансированными классами во всех моделях применён параметр `class_weight='balanced'`. Выбор оптимальной модели выполнен на основе метрик ROC-AUC и F1-Score. Обученная модель и параметры препроцессинга сохраняются в формате `.pkl`.

## Веб-приложение
Интеллектуальное приложение реализовано на фреймворке Flask. Представляет собой веб-сервис с HTML-интерфейсом для ввода параметров транзакции. При отправке формы данные проходят ту же предобработку, что и при обучении, после чего модель возвращает бинарный прогноз и расчётную вероятность мошенничества.


## Запуск проекта
1. Установка зависимостей:
   ```bash
   pip install -r requirements.txt
   ```
2. Обучение модели:
   Откройте `notebooks/fraud_detection.ipynb` в VS Code, выберите виртуальное окружение и выполните все ячейки. В папке `models/` будет создан файл `fraud_model.pkl`.
3. Запуск веб-сервиса:
   ```bash
   python app.py
   ```
   Интерфейс доступен по адресу: http://127.0.0.1:5000
